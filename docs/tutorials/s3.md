# S3 — Object Storage

Learn to create buckets, upload/download objects, configure versioning, lifecycle rules, and static website hosting with Ministack S3.

**Time:** ~15 minutes  
**Services:** S3

---

## Create and manage buckets

```bash
# Create a bucket
awslocal s3api create-bucket --bucket my-app-bucket --region us-east-1

# Enable versioning
awslocal s3api put-bucket-versioning \
  --bucket my-app-bucket \
  --versioning-configuration Status=Enabled

# Upload objects with metadata
awslocal s3 cp app.tar.gz s3://my-app-bucket/releases/v1.0/app.tar.gz \
  --metadata "version=1.0,environment=local"

# List with details
awslocal s3api list-objects-v2 --bucket my-app-bucket --prefix releases/
```

## Lifecycle rules

Set a lifecycle policy to automatically move old objects to cheaper storage tiers or delete them:

```bash
awslocal s3api put-bucket-lifecycle-configuration \
  --bucket my-app-bucket \
  --lifecycle-configuration '{
    "Rules": [
      {
        "ID": "archive-old-releases",
        "Status": "Enabled",
        "Filter": {"Prefix": "releases/"},
        "Transitions": [
          {"Days": 30, "StorageClass": "STANDARD_IA"},
          {"Days": 90, "StorageClass": "GLACIER"}
        ],
        "Expiration": {"Days": 365}
      }
    ]
  }'
```

Verify the policy was applied:

```bash
awslocal s3api get-bucket-lifecycle-configuration --bucket my-app-bucket
```

## Static website hosting

```bash
# Create a bucket for website
awslocal s3api create-bucket --bucket my-website --region us-east-1

# Enable website hosting
awslocal s3api put-bucket-website \
  --bucket my-website \
  --website-configuration '{
    "IndexDocument": {"Suffix": "index.html"},
    "ErrorDocument": {"Key": "error.html"}
  }'

# Upload content
echo "<h1>Hello from Ministack!</h1>" > index.html
awslocal s3 cp index.html s3://my-website/index.html \
  --content-type text/html

# Access via path-style URL
curl http://localhost:4566/my-website/index.html
```

## S3 event notifications to SQS

```bash
# Create a notification queue
awslocal sqs create-queue --queue-name s3-events
QUEUE_ARN=$(awslocal sqs get-queue-attributes \
  --queue-url http://localhost:4566/000000000000/s3-events \
  --attribute-names QueueArn \
  --query Attributes.QueueArn --output text)

# Configure S3 to send PUT events to SQS
awslocal s3api put-bucket-notification-configuration \
  --bucket my-app-bucket \
  --notification-configuration "{
    \"QueueConfigurations\": [{
      \"QueueArn\": \"$QUEUE_ARN\",
      \"Events\": [\"s3:ObjectCreated:*\"]
    }]
  }"

# Now upload a file — it triggers the notification
awslocal s3 cp index.html s3://my-app-bucket/test.html

# Check the queue
awslocal sqs receive-message \
  --queue-url http://localhost:4566/000000000000/s3-events \
  --query 'Messages[0].Body'
```

## Pre-signed URLs

```bash
# Generate a pre-signed upload URL (valid 1 hour)
awslocal s3 presign s3://my-app-bucket/upload-target.txt \
  --expires-in 3600

# Generate a pre-signed download URL
awslocal s3 presign s3://my-app-bucket/releases/v1.0/app.tar.gz
```

!!! tip "Tip"
    Pre-signed URLs from Ministack use `localhost:4566` — they work from any HTTP client on the same machine.
